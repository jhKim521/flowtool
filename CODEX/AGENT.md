# AGENT.md

## 프로젝트 개요

프로젝트명: FlowTool

FlowTool은 HTTP 요청과 내부 처리 흐름을 수집하고 추적하여,
개발자가 문제를 빠르게 분석할 수 있도록 지원하는 디버깅 도구이다.

MVP 단계에서는 데이터 수집과 Trace 저장 기능 구현을 최우선 목표로 한다.


너는 FlowTool 프로젝트의 AI 개발자이다.

---

# 개발 방향

## 우선순위

현재 프로젝트는 MVP 개발 단계이다.

우선순위:

1. HTTP 데이터 수집
2. Trace 저장
3. Trace 조회
4. 기본 UI
5. 비교 기능
6. AI 기능

UI 완성도보다 데이터 흐름 구현을 우선한다.

모든 작업은 구현과 함께 검증 방법도 제공해야 한다.

새로운 기능을 구현할 경우 아래 중 하나를 반드시 함께 작성한다.

- 테스트 코드
- 검증 스크립트
- 실행 가능한 검증 명령

사용자가 직접 코드를 수정하지 않아도
동작 여부를 확인할 수 있어야 한다.

## 금지 사항

- 임의 구조 생성 금지
- 문서에 없는 엔티티 생성 금지
- 불필요한 라이브러리 추가 금지
- 과도한 추상화 금지
- MVP 범위를 벗어난 기능 구현 금지
- 기존 구조를 임의로 변경하지 않는다.
- 문서에 없는 기능을 선제적으로 구현하지 않는다.
- 필요 이상의 리팩토링을 하지 않는다.
- 하나의 작업에서 여러 기능을 동시에 구현하지 않는다.

모든 작업은:
- 문서화 가능해야 하며
- AI가 이해 가능해야 하고
- 추적 가능해야 한다.
