# System Test Strategy

---

# 목적

System Test는

현재까지 구현된 기능이
실제 사용자 관점에서 정상적으로 동작하는지 검증한다.

Feature Verification과 달리

특정 기능이 아닌

FlowTool 전체의 사용자 흐름을 검증한다.

---

# 검증 대상

System Test는 다음을 검증한다.

- 기능 간 연결
- 사용자 흐름
- 데이터 정확성
- UI 동작
- Database와 화면의 일치 여부
- 사용성
- 개선 포인트

---

# 검증 범위

예시

HTTP 요청

↓

Capture 저장

↓

목록

↓

상세

↓

Database 확인

↓

사용성 확인

---

# 검증 원칙

System Test는

정상 동작 여부만 확인하지 않는다.

아래 항목을 함께 확인한다.

- 사용자가 목적을 달성할 수 있는가
- 기능이 자연스럽게 연결되는가
- 데이터가 일치하는가
- 사용하기 불편한 점은 없는가
- 개선이 필요한 부분은 없는가

---

# 수행 시점

예시

- 하나의 사용자 흐름이 완성되었을 때
- 여러 기능이 하나의 흐름으로 연결되었을 때
- MVP 단계가 완료되었을 때
- 사용자가 요청했을 때

---

# 결과

System Test 결과는

PASS / PARTIAL / FAIL

중 하나로 판정한다.

발견된 문제는

Issue로 기록하고

다음 Requirement 후보로 관리한다.
